## 🌟 Ключевые особенности

* ⚡ **Микросекундная точность:** Использование `QueryPerformanceCounter` и гибридного тайминга гарантирует честные 50 CPS без просадок.
* 💾 **Умное профилирование:** Встроенная СУБД **SQLite3** для хранения пресетов. База `macros.db` создается автоматически.
* 📊 **Живая статистика:** Отслеживание реального CPS и общего количества кликов за все время прямо в консоли.
* 🔊 **Аудио-отклик:** Звуковые уведомления через `Beep()` при включении, выключении и ошибках.
* 🛡️ **Защита от крашей:** Безопасное завершение потоков и сохранение данных в БД при экстренном выходе.
* 🎨 **SPA Консоль:** Перерисовка буфера без мерцания (Single-Page Application подход для CLI).

## 🚀 Быстрый старт

Проект настроен для моментальной сборки в современных средах разработки.

1. **Склонируй репозиторий:**
```bash
git clone https://github.com/твое-имя/aegis-macro.git

```


2. **Открой проект:**
Запусти файл `main.slnx` через **Visual Studio 2026**.
3. **Скомпилируй и играй:**
Просто нажми **`F5`** (Запуск с отладкой) или `Ctrl + F5` в Visual Studio. Программа скомпилируется и запустится. Все зависимости (включая исходники SQLite) уже вшиты в проект.

## ⚙️ Архитектура & Движок

Приложение построено по строгой трехуровневой архитектуре с использованием современных стандартов C++17 и WinAPI.

### 1. Гибридный тайминг (Backend)

Стандартная функция `Sleep()` в Windows имеет погрешность в 10-15 мс, что делает невозможным создание точного макроса свыше 30 CPS. В **Aegis Macro** реализован кастомный класс `PrecisionTimer`:

* **Грубый сон:** Если до следующего клика больше 2 мс, используется `Sleep()` для экономии ресурсов CPU.
* **Точный Spin-lock:** В последние доли миллисекунды поток переходит в режим активного ожидания, используя `_mm_pause()` (инструкция процессора для предотвращения перегрева ядра) в связке с `QueryPerformanceCounter`.
* **Результат:** Идеально ровные тайминги. Установив 50 CPS, макрос выдает ровно 49-51 физических кликов (через `mouse_event`).

### 2. Многопоточность

* **Main Thread:** Отрисовка UI и взаимодействие с пользователем (`std::cin` / `_getch`).
* **Clicker Thread:** Высокоприоритетный поток (`THREAD_PRIORITY_TIME_CRITICAL`), изолированный от интерфейса.
* **Hotkey Thread:** Асинхронный слушатель WinAPI (`GetAsyncKeyState`) для моментальной реакции на `F6`/`F7`.

### 3. База данных (Data Layer)

* **SQLite3** интегрирована на уровне исходного кода (`sqlite3.c`).
* Хранит пресеты пользователя и глобальный счетчик `__total_clicks__`.

## 🏆 Почему AEGIS? (Сравнение с аналогами)

| Фича | Обычные скрипты (Python/AHK) | AEGIS MACRO (C++) |
| --- | --- | --- |
| **Точность тайминга** | Плавает от 20 до 35 CPS | **Строго до 100 CPS** (QPC) |
| **Нагрузка на CPU** | Высокая (интерпретатор) | **< 0.1%** (гибридный sleep) |
| **Хранение настроек** | Текстовые .txt / .ini файлы | **Реляционная БД (SQLite)** |
| **Отклик интерфейса** | Зависает во время кликов | **Абсолютно независимый UI** |
