# Macro CLI - Трехуровневое консольное приложение

Проект представляет собой высокопроизводительный автокликер с профилированием настроек. Разработан в рамках экзаменационного задания с соблюдением трехуровневой архитектуры (Frontend, Backend, Database).

## Архитектура приложения

1. **Frontend (Presentation Layer):**
   Реализован в виде CLI (Command Line Interface). Обеспечивает легковесное взаимодействие с пользователем, не нагружая графическую подсистему ОС. Отвечает за маршрутизацию команд (выбор, создание профилей).

2. **Backend (Business Logic Layer):**
   Написан на C++. Ядро кликера вынесено в отдельный асинхронный поток (`std::thread`), взаимодействующий с WinAPI (`SendInput`) для минимальной задержки. Связь с фронтендом осуществляется через потокобезопасные атомарные переменные (`std::atomic`).

3. **Database (Data Access Layer):**
   Интегрирована реляционная СУБД **SQLite**. База данных (`macros.db`) инициализируется автоматически и хранит пресеты пользователя (имя профиля и значение CPS), обеспечивая персистентность (сохранность) данных между сессиями.

## Сборка и запуск
- Проект можно открыть через `main.sln` в Visual Studio (2019/2026).
- Библиотека SQLite встроена в проект (исходники `sqlite3.c` и `sqlite3.h` прилагаются), дополнительные SQL-серверы устанавливать не нужно.

## Управление
- Навигация осуществляется через меню в консоли.
- Горячая клавиша для запуска/остановки кликера: **Средняя кнопка мыши**.
